    API Reference

    This page is generated from the source by mkdocstrings. It documents every symbol exported from the top-level hodlbook package (its __all__).

    hodlbook

    hodlbook: a crypto paper-trading portfolio ledger built on pydynantic.

    AlertEvaluator

    Evaluates armed alerts against current prices and fires the crossers.

    evaluate(symbols)

    Fire every armed alert on symbols whose threshold the price crosses.

    Analytics

    Snapshotting, returns, and leaderboard ranking over a portfolio's value.

    leaderboard(limit)

    The top-limit portfolios ranked by total value, highest first.

    returns(portfolio_id)

    Percent change of total value from the first to the latest snapshot.

    Returns Decimal("0") when fewer than two snapshots exist or the first value is zero (no baseline to measure against).

    series(portfolio_id, *, cursor=None, limit=None)

    A most-recent-first page of value snapshots for a portfolio.

    take_snapshot(user_id, portfolio_id)

    Mark to market, persist a :class:Snapshot, and refresh the leaderboard.

    Raises :class:~hodlbook.errors.InvalidOrder (-> 422) if the portfolio does not exist -- the valuator does the existence check.

    An order was malformed (bad quantity/price, or unknown portfolio).

    MockPriceProvider

    A deterministic provider for tests and offline runs.

    Prices drift purely as a function of (symbol, drift_step) -- no random, no wall clock -- so a given step always yields the same price.

    Container of the five entity classes bound to one table.

    OrderExecutor

    Evaluates OPEN advanced orders against current prices and fills them.

    execute(symbols)

    Fill every OPEN order on symbols whose trigger the price has met.

    OrderNotFound

    Bases: HodlbookError

    An advanced order was requested by an id that does not exist.

    OrderStatus

    Bases: str, Enum

    Lifecycle state of an advanced order.

    OrderType

    Bases: str, Enum

    The kind of an advanced order.

    PriceCache

    A TTL cache over :class:PriceTick, fronting any :class:PriceProvider.

    PriceProvider

    Bases: Protocol

    A source of current USD prices keyed by symbol.

    Single method by design: a Protocol with only one required member avoids the default-method friction mypy raises when a Protocol mixes concrete and abstract members.

    RateLimitExceeded

    Bases: HodlbookError

    A principal exceeded the per-minute request quota (-> HTTP 429).

    Repository

    Persistence helpers for hodlbook entities, bound to one table.

    advance_dca(order)

    Persist a DCA order's advanced schedule, guarded on status == OPEN.

    Mirrors :meth:mark_order_filled -- the optimistic version lock plus the status == OPEN guard make a lost race a no-op. The caller mutates next_run / remaining_runs / status before calling.

    cancel_order(portfolio_id, order_id)

    Cancel an OPEN order, guarded so a cancel/fill race is safe.

    Guarded with F("status") == OrderStatus.OPEN: if the order has already been filled or cancelled, the conditional put raises :class:pydynantic.ConditionCheckFailedError, which we swallow and treat as already-resolved (the cancel is a no-op).

    create_order(portfolio_id, order_id, user_id, symbol, side, order_type, quantity, *, limit_price=None, interval_seconds=None, next_run=None, remaining_runs=None)

    Create an OPEN order, failing if one already exists at that key.

    create_portfolio(user_id, portfolio_id, cash=Decimal('0'))

    Create a portfolio, failing if one already exists at that key.

    delete_alert(portfolio_id, alert_id)

    Delete an alert by its primary key (idempotent).

    get_holdings(portfolio_id)

    All holdings for a portfolio (mirrors :meth:list_alerts).

    issue_api_key(user_id)

    Mint a new API key for user_id.

    Returns (raw_token, entity). The raw token is generated here and returned ONLY from this method -- the table stores its SHA-256 hash, so the plaintext cannot be recovered afterwards.

    list_open_orders_by_symbol(symbol)

    All OPEN orders for symbol across portfolios (GSI2).

    list_snapshots(portfolio_id, *, cursor=None, limit=None)

    Most-recent-first page of value snapshots for a portfolio.

    list_trades(portfolio_id, *, cursor=None, limit=None)

    Most-recent-first page of trades for a portfolio.

    list_trades_by_symbol(symbol, *, cursor=None)

    Most-recent-first page of trades for a symbol across portfolios (GSI1).

    mark_alert_triggered(portfolio_id, alert_id)

    Mark an alert triggered, conditionally so a double-fire is a no-op.

    Guarded with F("triggered") == False so a re-fire / lost race raises :class:pydynantic.ConditionCheckFailedError rather than silently re-stamping an already-triggered alert.

    mark_order_filled(order)

    Mark an order FILLED, guarded on status == OPEN AND version.

    The version lock is applied automatically by put from the loaded order's version; we add F("status") == OPEN so a concurrent fill/cancel makes this a safe no-op. Raises :class:pydynantic.ConditionCheckFailedError on a lost race.

    revoke_api_key(user_id, key_id)

    Mark a user's API key revoked (no-op if it does not exist).

    top_leaderboard(limit)

    Top-limit leaderboard entries by total value, highest first (GSI1).

    upsert_leaderboard_entry(user_id, portfolio_id, total_value, rank_key, taken_at)

    Insert-or-replace a portfolio's leaderboard entry at its primary key.

    The primary key is stable per (user_id, portfolio_id) so re-snapshotting overwrites the prior entry rather than accumulating; rank_key (the zero-padded value) rides along on GSI1 for ranked reads.

    Settings

    Bases: BaseSettings

    Process configuration, populated from HODLBOOK_* env vars / .env.

    Fields

    table_name: DynamoDB single-table name. aws_region: AWS region for the DynamoDB client. dynamodb_endpoint: Optional endpoint override (e.g. DynamoDB Local). price_provider: Price source -- "mock" or "http". price_ttl_seconds: Freshness window for cached price ticks. default_starting_cash: Default cash balance for new portfolios. log_level: Root log level (e.g. "INFO", "DEBUG"). rate_limit_per_minute: Max requests per principal per fixed minute window. metrics_enabled: Whether to record/serve Prometheus metrics.

    Side

    Bases: str, Enum

    Direction of a trade.

    SkippedOrder dataclass

    An order whose fill the engine rejected (insufficient funds/holdings).

    TradeConflict

    Bases: HodlbookError

    A trade lost the optimistic-lock race repeatedly and gave up retrying.

    TradeResult dataclass

    Outcome of a trade: the persisted trade plus realized P&L.

    realized_pnl is always Decimal("0") for buys (cost basis only moves on a sell) and (price - avg_cost) * quantity for sells. The Trade schema intentionally carries no P&L field; it is surfaced here instead.

    TradingEngine

    Executes atomic buy/sell orders against a :class:Repository.

    buy(user_id, portfolio_id, symbol, quantity, price)

    Buy quantity of symbol at price, atomically.

    sell(user_id, portfolio_id, symbol, quantity, price)

    Sell quantity of symbol at price, atomically.

    Valuator

    Computes a mark-to-market :class:Valuation for a portfolio.

    build_models(table)

    Define the five entity classes bound to table and return :class:Models.

    Defining the classes here (rather than at module scope) keeps the injected client isolated per table -- the same factory approach pydynantic uses in its own test suite.

    build_table(client, *, on_operation=None)

    Build the hodlbook :class:~pydynantic.Table bound to client.

    Pass on_operation to opt into pydynantic's observability hook -- it fires once per DynamoDB call for tracing and cost attribution. The default of None leaves behavior unchanged.

    collecting_hook()

    Build a hook that appends every event to a list, returned alongside it.

    Useful for tests and ad-hoc metrics collection::

    hook, events = collecting_hook()
table = build_table(client, on_operation=hook)
...  # events now holds one OperationEvent per DynamoDB call

    create_app(client, *, provider=None, clock=None, on_operation=None, settings=None)

    Build the hodlbook FastAPI app from an injected boto3 client.

    Services are constructed once here (never boto3 itself) and stored on app.state; the Depends accessors read them back per request.

    Pass on_operation (e.g. observability.logging_hook()) to enable optional tracing/cost-attribution logging around each DynamoDB call. The default of None leaves behavior unchanged.

    Pass settings to override the env-driven configuration (defaults to :func:~hodlbook.settings.get_settings); it drives values like the price cache TTL while leaving every other parameter backward-compatible.

    create_table(client)

    Issue a boto3 create_table for the hodlbook single table.

    PK/SK plus two GSIs (GSI1, GSI2), all attributes S, projection ALL, on-demand billing. Mirrors pydynantic's tests/conftest.py.

    get_settings() cached

    Return the process-wide :class:Settings, parsed once and cached.

    Call get_settings.cache_clear() after mutating the environment (e.g. in tests) to force a fresh read.

    logging_hook(logger=None)

    Build an :data:~pydynantic.OperationHook that logs each operation.

    Each :class:~pydynantic.OperationEvent is logged at DEBUG on success and at INFO on failure (the exception name is included). Pass a custom logger to route the records; defaults to logging.getLogger( "hodlbook.dynamodb").

    setup_logging(level)

    Configure the root logger with a single JSON-emitting stream handler.

    Idempotent across calls: any handler previously installed by this function is removed first so repeated create_app calls (e.g. in tests) do not stack handlers. level is a stdlib level name such as "INFO".
